finding key codes to use for bindkey

September 08, 2013 [ hack ] #shell Read in 2 min.

You can find keys in shell using .inputrc or .screenrc. In order to find the key codes to use, you can use either read command or press C-v and then enter in the key you like to map.

I have the following code in my .bashrc to figure out how keys are mapped. It excludes mapping with keywords which we are not interested with:

alias bindp='bind -p | grep -E -v "vi|bound| self|lowercase|argument"'

bound only shows up if the key is not bound self prints itself, so it tends to be important lowercase is for commands that is immune to shift key argument is for ???

Sometimes keys do not work

Binding special keys sometimes will not work, because the system grabs it before a terminal can. There is a work around so that the control is given up. In the following example, and are mapped to cycle through windows in screen. This works for xterm, unfortunately this does work not for gnome-terminals (including terminator) for the aforementioned reason.

Add the following lines to .Xresources:

*vt100.translations: #override \n\
    Ctrl ~Shift <Key>Tab: string(0x1b) string("[27;5;9~") \n \
    Ctrl Shift <Key>Tab: string(0x1b) string("[27;6;9~") \n

reload .Xresources:

xrdb ~/.Xresources

and add the following lines to .inputrc:

# Ctrl-Tab
bindkey "^[[27;5;9~" next

# Ctrl-Shift-Tab
bindkey "^[[27;6;9~" prev

to use if-statement in .inputrc, the syntax is as follows

$if mode=vi
$endif

to reload .inpturc:

bind -f  ~/.inputrc

inputrc is originally read by readline. Its complete list of option is on a ss64 page
