SSH recipes
January 25, 2014 [ sysadmin ] #shell Read in 2 min.

SSH keyless entry does not work

Check the logs on /var/log/auth/log or /var/log/secure (CentOS). You probably have a permission issue.

chmod 700 ~/
chmod 700 ~/.ssh/
chmod 600 ~/.ssh/authorized_keys

make sure the options are set properly on /etc/ssh/sshd_config - it needs root permission

RSAAuthentication yes
PubkeyAuthentication yes
AuthorizedKeysFile      %h/.ssh/authorized_keys

then restart

sudo service sshd restart

Diff files over remote

ssh <remote-host> "cat /path/to/remotefile" | diff - /path/to/localfile

Copy over SSH key so you don't have to type password every time:

sshk() {
    ssh-copy-id -i ~/.ssh/id_rsa.pub "$@"
}

Add keys to ssh so that it is used on certain git repositories:

in .ssh/config

Host work.com
HostName work.com
IdentityFile ~/.ssh/work_rsa

then add the key to ssh ssh-add ~/.ssh/work_key

Reverse Tunnel (Login)

Run the following command on a machine (fireWalledServer) you want to run

ssh -NR 9999:localhost:22 user@78.12.255.255

SSH from your local machine:

ssh localhost -p 9999

Reverse Double Tunneling

Assuming that reverse tunnel is set up from the previous instruction, access the firewalled server from 3rd machine

ssh user@78.12.255.255

in another shell, run:

ssh localhost -p 9999

Reverse Tunnel (non-login, port forwarding)

ssh -nNT -R 8000:local.mydomain.com:9999 remote.mydomain.com Open port 8000 for remote.mydomain.com and send it to local.mydomain.com:9999 - The address can be localhost:9999 instead for local routing. - the address before 8000 is ommited , but it can be specified if there are multiple NICs that can access the internet. -n associate standard input with /dev/null -N do not prepare the command stream -T do not allocate psuedo-tty on the remote system

Skip key checking

This makes you susceptible to Man-in-the-middle attack. I only use it work where the keys change consistently.

Host *
    StrictHostKeyChecking no
    UserKnownHostsFile=/dev/null

Speed up SSH login in

Host *
    UseDNS no
    GSSAPIAuthentication no

or add those lines to /etc/ssh/ssh_config for system change.

Execute the command in the background

ssh -n -f user@host "sh -c 'cd /whereever; nohup ./whatever > /dev/null 2>&1 &'"
