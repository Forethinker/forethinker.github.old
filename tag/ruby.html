How to get Ruby, RVM, Jekyll, Octopress, and Heroku working on Cygwin
August 08, 2013 [ hack ] #ruby, #cygwin Read in 3 min.

Warning: I do not use Ruby frequently enough and the following has been an exercise for me to just test Octopress and Jekyll.

Installing Ruby

ruby can be installed easily on cygwin via setup.exe or apt-cyg, but gem that comes with it seems to be outdated and does not work properly. This can be easily fixed with gem update --system. The dependencies, however, must be met by Cygwin installation. Make sure to install 'wget', 'gcc', and 'makefile'.

Installing RVM

RVM has dependency of 'patch' and its installer needs to be modified because it fails the bash version check. Matt Scharley has the instruction, but it did not work for me right away. You can get my rvm-installer, but really it is just the file from the repo with the first paragraph after #! removed.

cat rvm-installer | bash -s stable --ruby
source ~/.rvm/scripts/rvm

Installing rake-compiler, POSIX-spawn, jekyll

This is the continuation of Matt Sharley's instruction, with a bit of fixing.

gem install rake-compiler
git clone git://github.com/rtomayko/posix-spawn.git
cd posix-spawn/
gem install pkg/posix-spawn-0.3.7.gem
rake gem
gem uninstall posix-spawn-0.3.6
gem install jekyll
easy_install Pygments

Installing Octopress

gem install bundler
cd octopress
# Modify the following line on Gemfile.lock
# gem 'posix-spawn', '~> 0.3.7'
# Modify the following lines on Gemfile
# gem 'rake', '~> 10.0.4'
# gem 'json'
rake generate
rake deploy

Installing Heroku

For heroku toolbelt you need a custom install script instead of the one provided on the heroku website. Copy the code below, save it as cygwin-install.sh. Make it executable chmod +x cygwin-install.sh and run it ./cygwin-install.sh.

HEROKU_CLIENT_URL="http://assets.heroku.com.s3.amazonaws.com/heroku-client/heroku-client.tgz"

rm -rf /opt/heroku
mkdir -p /opt/heroku
cd /opt/heroku

if [[ -z "$(which wget)" ]]; then
curl -s $HEROKU_CLIENT_URL | tar xz
else
wget -qO- $HEROKU_CLIENT_URL | tar xz
fi

mv heroku-client/* .
rmdir heroku-client
ln -s /opt/heroku/bin/heroku /usr/bin

The problem is that the windows version does not translate cygwin's path (/cygdrive/c/...) properly into windows version (c:\...). So you have to use either Ubuntu or Standalone version. But wait, their install scripts fail because cygwin does not use either apt-get or sudo. Strip those out and modify a bit so that the heroku binary is in the path. I did the modification for my preference. If you are looking for foreman you can install it by gem install foreman

Run your first Heroku app

Go to the repository where the files are waiting to be deployed.

# type in your credentials
heroku login
# create your public and private key. Skip if you have them already
ssh-keygen -t rsa
# add your keys to heroku account so that you don't have to type in the username and password
heroku keys:add
# create git repo if it does not exist and link it to a heroku app
heroku create
# deploy
git push heroku master
