vSphere Client Quirks
December 13, 2013 [ dev ] #cloud Read in 3 min.

vSphere Client is a client program to monitor and configure VMWare cluster. The configuration options are significantly lacking compared to VMWare Station (Paid) or VMWare Player (Free), a software for the local virtualization. I cannot comment on all features, but I will list some that I thought were lacking

VMWare Tools

VMWare Tool allows a easier VM shutdown, directory/filesystem sharing, clipboard sharing, and much more. VMWare tool installation is a breeze on Station and Player. I had to do manual installation with vSphere.

Clipboard Sharing (fixed)

I had to go to Options->Advanced->Configuration Parameter and add following options in order to get clipboard sharing to work

isolation.tools.copy.disable false
isolation.tools.paste.disable false

Ctrl+Alt+Shift

I am a power user and ctrl+alt mappings in my linux setup. I immediately change the default hotkey to ctrl+alt+shift. There is no way to change this in vSphere, not even in Configuration Parameter. If you think I am wrong you can try putting the following lines, but it won't work.

pref.hotkey.shift = "true"    
pref.hotkey.alt = "true"    
pref.hotkey.control = "true"
pref.hotkey.gui = "false"

The only workaround is the following: With the guest having the keyboard/mouse focus, you press Ctrl-Alt-Space, release Space (still press Ctrl-Alt), then hit the rest of the shortcut key. (e.g. F1). I am a bit frustrated but I appreciate the workaround.

Repeated Characters (fixed)

Because of the network delay, sometimes I got a repeated key press. This is because while I am waiting my VM to respond, VM is actually running and sees my action in a frozen state. The following assumes that my connection speed will be up to the threshold within 2 seconds.

keyboard.typematicMinDelay = "2000000"

The delay is in micro seconds

Full Screen Mode Behavior

When you are in a full screen mode, there is no way to minimize the screen. Also it is set to "Always-on-top" mode, which gets pretty inconvenient when

Conclusion

All these problems go away if you use VMWare Workstation to view the VM on a local machine. It is a shame though, that VMWare is making money out of these features which should I think should be given freely for user-preference.

P.S.

This may seem a out of space, but I found the DPI settings of Gnome to break sometimes due to VMWare's auto-resolution function that kicks in when maximizing the screen. In this case running the following command helps to fix the problem:

gnome-settings-daemon
