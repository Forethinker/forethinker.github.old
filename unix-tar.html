Unix Tar

February 14, 2014 [ dev ] Read in 1 min.

Like ln, the file that gets creates come first. This for me is confusing. I expect all linux to follow pattern, but free-rider can't be a chooser, I guess.

Another thing that bugs about tar utility is that you have to specify the extension and tar does not do it for you automatically. This probably probably has wasted a lot of time over decades.

using . as will unpack files to current directory when uncompressed. This is kind of a naughty behavior if you are distributing your files to public, and especially so if there are lots of files. If you don't include the top directory in a tarball but want to specify some other path, -C does the job.
