Intuitive and Natural Windows Management Setup

June 06, 2013 [ hack ] #windows Read in 6 min.

This is the final result after five years of searching for an efficient windows management system that works with 30+ instances of windows.

The Search

The first windows manager that I tried beyond the basic one supplied on OS was Awesome Windows Manager in Linux Mint. AWM is a titling windows manager, so that you can use the most out of the screen estate without having to resize windows every time. Combined with multiple workspace, you could move and group windows without too much hassle. I liked AWM so much that I installed it also on Ubuntu when gnome2 vs 3 was a hot debate and I also sought a similar program in Windows. Seek and ye shall find. Bohold, bug.n windows manager! I customized every bit so that I could manage windows with my just keystrokes It also very good except for two things. It did not support dual screen very well, at least for me. Although I am now an avid single screen user, I used to rely on dual screen when working on a complex project. It was a bit buggy, but the real deal breaker was that the screens would get jumbled and not work properly when I change the connectivity to or from an external monitor. I do this frequently because I have a dedicated dock for my laptop. The only way to fix once the problem persist was to reboot. It really cut the flow of my work and I could not use it anymore.

After I parted with tiling windows managers, my afterthought was not so positive:

  • I found myself spending too much time to organize the windows. My latest attempt at getting around this problem was organizing workspace by similar instances of windows automatically, but it did not work so well when I needed to see different applications side-by-side.
  • It is quite burdening on my memory to keep track of which windows are where.
  • I kept on changing my customization, thinking that it will improve my productivity.
  • It can be super annoying when you just want to minimize or maximize certain windows. Yes, you can do this, but then it is not easy to switch back to way things were.
  • Maybe I have a bad impression because I struggled long time with windows version of tiling windows manager, but nonetheless, I wanted a solution that worked for Windows.
  • Or maybe I am a human, I am used to moving objects around in a physical world. Things are intuitive if they are natural. Things are natural if there is a pattern.

I kept on searching though. And I came to conclusion that Windows has something right. Of course, in terms of customization, anything is possible in Linux so it is definitely possible to mimic the functionality, but Windows 7 and above support what I need out of box and its functionality can be extended using Autohotkey. I added a workspace program Dexpot and I had a winner.

Windows + Autohotkey

So now my setup is to use Windows default keyboard shortcut of Win+NumKey. Here are the benefits

  • You could rely less on Alt-tab, which gets very inefficient when you have a lot of applications open
  • You could cycle the windows for one specific application with multiple Win+NumKey
  • You could open a new instance of that application with Win+Shift+NumKey (I replaced this feature so that I could map more applications; see below)
  • If you disconnect the external monitor, the windows will still be half screen position regardless of the resolution if you assigned it using Win+Arrow key. This is a big win for me.

Here are the limitations out of the box and how I worked around it. The maximum number of applications you can assign with shortcut keys are only 10. I frequently use about 20 applications and I typically need one instance (one window) for each of those applications. So I assigned Win+Shift+NumKey to other applications.

#+1::
SetTitleMatchMode, Regex
IfWinExist Chrome
{
    IfWinNotActive Chrome
    {
        WinActivate
    }
    Else 
    {
        Send #{Down}
    }
}
Else
{
    Run C:\Program Files (x86)\Google\Chrome\Application\chrome.exe
}
return

This autohotkey script allows you to minimize the window if is already open, open a new instance otherwise. I had to give up The option to cycle through, but I actually prefer this option. - You still have to use arrow keys and arrow keys mean you move you hand out of the position. So I mapped it to vim-like motion keys as follows

#h::Send #{Left}
#k::Send #{Up}
#j::Send #{Down}
#l::Send #{Right}

;------------Lock Screen Key------------
#ScrollLock::
RegWrite, REG_DWORD, HKEY_CURRENT_USER, Software\Microsoft\Windows\CurrentVersion\Policies\System, DisableLockWorkstation, 0
Sleep, 500
DllCall("LockWorkStation")
Sleep, 2000
RegWrite, REG_DWORD, HKEY_CURRENT_USER, Software\Microsoft\Windows\CurrentVersion\Policies\System, DisableLockWorkstation, 1
return

You will have to do registry edit in order to prevent the screen from locking with win+l. I also remapped Win+Scrolllock to lock the screen. Here is to disable screen lock:

Windows Registry Editor Version 5.00

[HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Policies\System]
"DisableLockWorkstation"=dword:00000001

Here is to enable screen lock:

Windows Registry Editor Version 5.00

[HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Policies\System]
"DisableLockWorkstation"=-

You can download them here.

Here are other design choices I made:

  • I would like to think shift as a key for an alternative option. So for example. If 1 is assigned to firefox, ten I would assign shift+1 to Google Chrome.
  • Assign most frequently used applications to 1-6. It just felt natural to me this way. Also since you are using right hand to control the window position, you will find this to be more efficient setting in the long run.
  • If you want more applications mapped, I suppose you could go down one line staring with QWERT.

Dexpot to Finish It Off

I mapped keys for managing workspace for Dexpot as follows:

W+F1 Workspace 1
W+F2 Workspace 2
...
W+Shift+F1 move the window to workspace 1
W+Shift+F2 move the window to workspace 2
...
W+Tab switch workspace (most recent)
Ctrl+Alt+Arrow go to workspace in that direction
Ctrl+Alt+Shift+Arrow move the window to the workspace in that direction

Bringing the result back to Linux

Now I am setting openbox's shortcut in Cruncbang linux OS to be something similar to above, including switching to already open application]. I do not use my linux partition as often, but once I do, I will upload scripts.
