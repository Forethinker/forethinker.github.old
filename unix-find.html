Unix Utility Find

December 02, 2013 [ dev ] #shell Read in 3 min.

Default

  • breadth-first-search
  • search starts from 24 hours ago
  • search the current directory
  • -P: never follow symlink (Default)
  • * match any characters

Patterns

Each option has optional prefix -i for case-insensitivity - name - regex - path

Numbers

  • inum: inode number
  • links: number of links
  • gid: group ID
  • uid: user ID

Attributes

  • mount: do no search through mounted directories (CD, USB, remote drive)
  • nogroup
  • nouser
  • readable
  • empty
  • exectuable

Others

  • samefile
  • user

size

Negating Search Query

  • ! and -not does the same thing, except that -not is POSIX-compliant. The expression that comes after is affected and it needs to be escaped \( ... \)

Expression

  • Expressions are logically ored with -o and space is implied and -a

types (first letter is the option)

  • block, character, directory, pipe, file, link, socket

file comparison

  • newer: modified time than the specified file
  • newerXY: XY can be one of the prefix below the argument is the filename to use for comparison

time

prefix

c - changed
a - accessed
m - modified

root

min - minutes
time - days (24 hours)

space + number

+ greater
- less than
exactly otherwise
  • used: accessedTimes + n*days > modifiedTime
  • anewer: accessedTime > modifiedTime
  • cnewer: changedTime > modifiedTime
  • daystart: measure time from the beginning of today

Actions

  • delete (implies depth)

  • prune: do not descend into driectory, cannot use with depth or delete

    find [path] -type d [conditions to prune] -prune -o [your usual conditions] [actions to perform]

  • without -typd d, prune will not print the pruned files, but will still print out the directories containing them. stackoverflow

  • prune is delimited by -o for its conditional expressions, except when when it is preceded directly next to -o.

Actions (-ls)

if the flag is prefixed with f, a fileName for output is to be passed as the argument to the flag. - ls: - print: fullName - print0: fullName appended with a null character - printf: formatted output

Actions (-exec)

the argument is any executable in the path execdir: excutes the command on the subdirectory where the matching file was found ok: exec with confirmation okdir: execdir with confirmation "{}": is alias for the matched file name(s). Use double quotes for possible spaces between names \;: execute the command on each matched file \+: append all file names then pass to the command

only shows the line numbers

find . -exec grep -Hn hello {} \;

show also the filename

find . -exec grep -n hello /dev/null {} \;

Depth

  • it is better to include it early in the options so that you don't get a warning
  • depth: process each directory's contents before the directory itself (Depth First Tree)
  • maxdepth: sub-directory level 2 and 4 -mindepth 3 -maxdepth 5

symlinks

  • L: follow symlinks
  • H: do not follow symlink except for command line argument processing

Permission

  • perm: permission

regex

  • follows the syntax of grep: special characters needs to be escaped and
