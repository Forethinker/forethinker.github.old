OpenVPN Notes

October 10, 2013 [ hack ] #security Read in 3 min.

Running OpenVPN with an alias

OpeVPN configuration comes in many different files: .crt, .key, .ovpn, .key. If you cd into the directory containing these files and sudo openvpn ***.ovpn, you are good to go. I wanted to set up an alias for this, omitting cd part. The trick is to add --config

sudo openvpn --config /path/to/myopen.ovpn

Let's say you have a password file associated with your OpenVpn connection. You would save your username and password to some file , say password.txt and store it along with other files. With the previous command, OpenVPN will complain and fail to find that file. You need to supply --cd flag in this case, which changes the initialization directory. 

sudo openvpn --config /path/to/myopen.ovpn --cd /path/to/

Just supplying --cd will not work for the first case. And here is my final script

sudo openvpn --config ~/Dropbox/server/openvpn/server.ovpn --cd ~/Dropbox/server/openvpn/ 2>&1 > /dev/null &

Windows Installation

On a Windows Machine, I was keep getting this message after installing OpenVPN GUI.

"All TAP-Win32 adapters on this system are currently in use."

Apparently despite this ensuring message: "Installation Package (Both 32-bit and 64-bit TAP driver included):" on the download site. The driver was not getting installed properly. After downloading the .exe from from OpenVpn Website, the drivers were properly installed and I was able to use OpenVPN GUI again.

tap vs tun

In the .ovpn file there is always a line that starts with dev, which is short for device. tap allows you to see the VPN while not losing your local IP subnet, effectively Ethernet becomes another ethernet switch for you. This introduces some overhead on the packet and you will also pick up the broadcasting messages. But using tun goes layer lower (layer 3) and routes all your traffic via VPN. If you are illegally downloading something, tun is not the way to go.

Password-less OpenVPN

Save a file with the following content

username
password

and save it with whatever you like (but the extension is important) in the same directory as your *.ovpn file. In the *.ovpn file, add a line:

auth-user-pass password.txt

This works in Linux, but not in Windows, because Windows client is not compiled with the right option by default. It is a good security measure considering the high percentage of Windows machines getting compromised.

Setting password on Key file with OpenVPN GUI

You have the option to enable password on *.key in addition to authentication. This can be done by right clicking the icon and clicking `Change Password, with OpenVPN GUI, once you set it, you cannot unset it. It will force you to type another password of at least 8 characters if you try to change it. You will either have to decrypt it on a Linux machine or just copy the saved copy of Key file over.

OpenVPN also does not allow tabbing through the prompt but it allows shift-tabbing.

DNS server not getting pushed on Windows

Just add this line to your *.ovpn file:

push "redirect-gateway"

or to push your own DNS server:

push "dhcp-option DNS 10.3.1.11"
