Installation

Ubuntu has node package, which makes things interesting. Many of the npm modules likes to refer node, but the actual nodejs binary is installed as nodejs. First you have to make sure you don't install node with apt-get, second you have to make a symlink nodejs to node.

sudo ln -s /usr/bin/nodejs /usr/bin/node

Phpstorm

Pycharm (3.1.3) does not work with LiveEdit because it does not have a native web server. I just figured that for anything web-related Phpstorm is going to edge over Pycharm. The watcher feature in Phpstorm is really cool. The documentation shows how to use it with coffeescript.

The show two examples to source and output directory resolution, but I think that going to be what you need.

Arguments: --bare --output $ProjectFileDir$\js\ --compile --map $FileName$

$ProjectFileDir$\output\$FileNameWithoutExtension$.js:$ProjectFileDir$\output\$FileNameWithoutExtension$.map

--bare --output $ProjectFileDir$\js\$FileDirRelativeToProjectRoot$\ --compile --map $FileName$
$ProjectFileDir$\js\$FileDirRelativeToProjectRoot$\$FileNameWithoutExtension$.js:$ProjectFileDir$\js\$FileDirRelativeToProjectRoot$\$FileNameWithoutExtension$.map

I have added --bare, because by default coffeescript wraps the output in an immediate function. Immediate functions will provide a namespace protection for the javascripts you concatenate or mingle and it is considered a good practice.

(function() {
    Yourcode
})();

So what I should be doing is binding the javascript function to an event handler instead of calling it directly:

document.getElementbyId("theButton").addEventListener('onclick', testing, false)

If you feel like a Darth Vader who needs more force, there is a javascript tool called see.js to evaluate these nested javascript scope.

Also don't forget to add an Extra line of return so that the controller or the module does not explictly return anything. This actually could break the functionality of the web application.

Syntax

"#{}" evaluate the expression within {} before stringifying it. having an explicit return statement at the end of a function or having (-> ) splat operator. avoids having any value returned.

use Casper#evaluate() to access actual page DOM and manipulate elements.

casper.start('http://google.fr/', function() { require('utils').dump(this.getElementInfo('#hplogo')); });
