How to install pip, oauth2 on Windows

May 10, 2013 [ dev ] #python Read in 3 min.
  • pip is like apt of debian-based system for python. It installs and uninstalls packages
  • distribute is like dpkg of debian-based system for python. It builds and distribute packages.
  • In short, they make installing packages for python much easier. For a longer explanation, here is a stackoverflow answer, and the official description can be found the python website
  • Some people might be using setuptools in windows, but it is outdated. setuptools is not available for Windows 64-bit platform anyway, so the choice is limited for many people out there.
  • distribute replaces setuptools starting from python3, but it is available for python2 as well

Native Windows Installation

  • Download Distribute and Pip from the following website.
  • Make sure you get the right version and platform for your python (i.e. 32 vs 64 bit, 2.7 vs 3.2).
  • Just click buttons to finish installation, unless you installed Python somewhere other than C:\
  • In the command prompt or Cygwin, run the following:

 C:\Python27\Scripts\pip.exe install oauth
 /cydrive/c/Python27/Scripts/pip.exe install oauth

Cygwin Installation

  • First use Cygwin setup to install Python for Cygwin.
  • download and run the following file
  • easy setup downloads the package for you and installs it, but it is based on setuptools and pip is recommended.

python ez_setup.py
  • now you will have easy_install in your excitation path and you can install pip and distribute

easy_install distribute distribute pip
  • now you can install anything via pip as you would in other environment

pip install oauth

[Cygwin installation Errors]

If you see an error like this when you install a package with pip

ImportError: No module named _io

Make sure you set your Anti-Virus program so that it doesn't delete files in cygwin directory. Then reinstall python with cygwin and try again.

Linux Installation

I first did

sudo apt-get install python-setuptools python-dev build-essential python-pip
pip install oauth

and the installation was successful, but it did not run properly. I got the following error

$ python twitterstream.py 
Traceback (most recent call last):
  File "twitterstream.py", line 1, in <module>
    import oauth2 as oauth
  File "/usr/local/lib/python2.7/dist-packages/oauth2/__init__.py", line 32, in <module>
    import httplib2
  File "/usr/local/lib/python2.7/dist-packages/httplib2/__init__.py", line 1318
    for key in _get_end2end_headers(response):
                                         ^
SyntaxError: invalid syntax

The reason seems to be that python3's httplib2 was getting installed instead.. So I uninstalled httplib2:

sudo uninstall httplib2

and download the file manually

tar -xvzf httplib2-*.tar.gz
cd httplib2*

and run the following commands in the folder 

python setup.py build
sudo python setup.py install
