Zsh + Oh-My-Zsh

In the order of awesomeness: - fast - remote auto-completion: scp serverName:<tab> you can have auto-completion on your server. - faster completion compared to bash_completion - auto-correction of commands - easy customization with oh-my-zsh. Many themes are available. - substantial community support and active development - extended globbing - shared history (out of the box) - improved path expansion - variable expansion: $PATH<tab> - Extending globbing: **/*.c matches directories recursively; zmv '(*).mp3' '$1.wma' renames extension - suffix alias: alias -s c=vim => ./program.c now opens the file in vim - argument history traversal with Alt-/

Fish

In the order of painfulness: - slow - not compatible wish bash - development Pace is also slow - pOSIX non-compliant

Bash: I kind of see zsh as the extended version of bash, but I still found zsh to be faster. Here is one example of the loading time. ls speed was comparable to bash Timing from bash:

$ time bash -c exit
real    0m0.282s
user    0m0.060s
sys     0m0.180s

$ time zsh -c exit
real    0m0.040s
user    0m0.000s
sys     0m0.031s

Timing from zsh:

time bash -c exit
bash -c exit  0.09s user 0.14s system 78% cpu 0.288 total

time zsh -c exit
zsh -c exit  0.00s user 0.03s system 93% cpu 0.033 total
