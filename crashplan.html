Installing Crashplan on Pogoplug

April 22, 2013 [ hack ] #server, #backup Read in 4 min.

I am assuming that you have Arch Linux Arm running on your Pogoplug. My pogoplug is V2, which has Marvell processor in it. (1.2 Mhz). Here is an excellent article by Obihoernchen that helped me to get things set up properly.

Installing Java SE

Download the one that starts with "ARMv5 Linux" from the Oracle Website. Unfortunately, You will have to sign up for Oracle account and upload the file to your server via SCP:

# cd /path/to/your/file
# nas is the alias defined in .ssh file for my server. Change 'nas' with your server's ip address
scp ejre* nas:/usr/lib
ssh nas
cd /usr/lib
tar -xzvf ejre*
rm -rf ./ejre*
# create a symlink to the java binary
sudo ln -s /usr/lib/jvm/bin/java /sbin/java
# Check if your java works; you might have to logout and log back in for the PATH to refresh
java -version

Install Crashplan

wget http://download.crashplan.com/installs/linux/install/CrashPlan/CrashPlan_3.4.1_Linux.tgz
tar -xzvf CrashPlan_3.4.1_Linux.tgz
./install.sh
# choose the default for all options
cd /usr/local/crashplan
# -p is to preserve the file attributes
cp -p linbjtux.so{,.orig}
wget http://www.opticality.com/blog/wp-content/uploads/files/libjtux.so
cd /tmp
wget http://ftp.us.debian.org/debian/pool/main/libj/libjna-java/libjna-java_3.2.7-4_armel.deb
ar vx libjna-java_3.2.7-4_armel.deb 
tar -xzf data.tar.gz
cd /usr/local/crashplan/lib
mkdir -p com/sun/jna/linux-arm
cp /tmp/usr/lib/jni/libjnidispatch.so com/sun/jun/linux-arm/
cp -p jna-3.2.5{,.orig}

libjtux.so was created by hadar, but just in case, I have my copy here.

Setting up systemd

wget http://4thinker.com/files/crashplan
cp crashplan /etc/rc.d/

wget http://4thinker.com/files/crashplan.service
cp crashplan.service /usr/lib/systemd/system/

Test via systemctl

systemctl enable crashplan
#check the status of crashplan
systemctl status crashplan

Log in from the Crashplan client GUI

On you computer find this file: 'c:\Program Files\CrashPlan\conf\ui.properties' and uncomment the following line

servicePort=4200

And open the SSH tunnel to your server's 4243 port which services Crashplan by default:

ssh -N -L 4200:localhost:4243 username@servername

Launch crashplan and its head should read data from the server. You can check this by looking at the name of the computer on the program. I actually have the following line in my .bashrc so that I could set up the tunnel and launch the crashplan at the same time. '&', runs the command in the background.

alias crashplan='ssh -N -L 4200:localhost:4243 nas& "$(cygpath "C:\Program Files (x86)\CrashPlan\CrashPlanDesktop.exe")" '

I also added these lines in my crontab so that crashplan does hog the network during the peak times. You can edit crontab with crontab -e which opens up the file in your default editor (as defined by $EDITOR) and see the content of crontab with crontab -l in your commandline.

@daily systemctl start crashplan
0 14 * * 1-5 systemctl stop crashplan

And you should be good to go!

Other JDKs and their shortcomings

openjdk8 does not work becuase it does not have a support for ARMv5 yet openjdk7 does not work because there is a bug in the interpreter. I get the follow message:

 Error: caught unhandled signal 4
 Error: caught unhandled signal 11
 etc...

jre6 works, but it is a bit outdated and has poorer performance. (https://blogs.oracle.com/henrik/entry/java_se_embedded_7u2_released)
