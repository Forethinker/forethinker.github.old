-i Case insensitive match. -c Count of matches; suppressing normal output. One line per file match. -o Show only the part of a matching line that matches. -v not -e or

-a processes binary file. -f read the matches from a file. -o show only the matched portion. -z a line is terminated by null byte instead of a new line character.

grep -E extends the functionality of grep, allowing the usage of ?, +, {, |, (, and ) wihtout having to have a blackslash in front of them. grep -F is a fixed string and matches all characters literally. grep -P uses experimental perl-mode, but pcregrep is a better choice. egrep is deprecated and grep -E is favored. fgrep is deprecated and grep -F is favored. Older systems needed these binaries.

AND

and can be achieved in two ways, by using exteneded pattern or by chaining the two grep operations together.

grep -E 'pattern1.*pattern2|pattern2.*pattern1' filename
grep -E 'pattern1' filename | grep -E 'pattern2'

basic - escaped the special characters: \?, +, {, \|, (, ), grep -E does the opposite, and uses [{] to match literal {}.
