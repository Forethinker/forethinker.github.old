Shougo's Neocomplcache + Neosnippet + Vimproc + Vimshell

September 16, 2013 [ hack ] #vim Read in 3 min.

I am very reluctant to write about my vim configuration, because it changes so frequently. But I really using vim plugins by Shougo, I could not help writing something about it.

Neocomplcache

IMHO, the best autocompletion plugin around. I use the generic configuration as shown on the github page, with exception to two lines.

let g:neocomplcache_dictionary_filetype_lists = {
\ 'javascript' : expand('~/Dropbox/dev/dotfiles/others/javascript.dict') .','. expand('~/.vim/bundle/vim-node/dict/node.dict'),
\ }

let g:neosnippet#snippets_directory='~/.vim/bundle/vim-snippets/snippets, ~/Dropbox/dev/dotfiles/snippets'

The first line allows me to add dictionaries that are not part of the installation. One is linked to my Dropbox and the other to another vim plugin for javascript auto-completion. The second line is used to add other snippets. I just use Honza's snippets and my custom snippets, once again it is synced using Dropbox. It's very nice this way.

NeoSnippet

NeoSnippet shines over Honza's snippet because it displays the snippet signature in the autocompletion box. I add Honza's snippet to the mix anyway, because there are some that are not included in Neosnippet. I actually do not use snippet very often, but it is still a nice to have. Neosnippet works with Neocomplcache out of the box.

Neobundle

I only installed it because I thought it would simplify my vimshell installation. But I found it slower than Vundle, so I am going back.

Vimproc + Vimshell

Vimproc is kind of like an add-on for many of Shougo's newest inventions. I only installed it for vimshell and neobundle. It is a little tricky to install. Someone wrote that installing NeoBundle will compile Vimproc automatically. That was not the case for me in Windows. There are many options to install in Windows: mingw, msvs (MS Visual Studio), or cygwin. Of course, I went on with cygwin.

cd ~/.vim/bundle/vimproc/
make -f make_cygwin.mak

This will allow you to run Vimshell in Windows. But I still had trouble running Vimshell in gVim that I use, which is compiled to run on Windows natively using mingw-w64. It may seem reasonable to install mingw-w64 version of vimproc by installing mingw on cygwin. I found out that I did not have to do this. You can try if you'd like, but you still have to download this file a .dll file from this page to get the compilation going. If you use Windows gVim like I do, that file all you need. Just move it ~/.vim/bundle/vimproc/autoload and you are all set. Having cygiwn inside gVim is very nice. Here is a more detailed how-to for installation.

Watch Out for Capitalization

The capitalization matters when you specify the plugins to install. Shougo is capitalized.
