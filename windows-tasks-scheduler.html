Using Windows Tasks Scheduler

July 11, 2014 [ hack ] #automation Read in 2 min.

I hardly ever use Windows Tasks Scheduler. Because I spend most of my time on computer within a few selected applications, I tend to forget about it. Well, setting once and forgetting is the whole point of task scheduling, I find myself updating things constatntly, and when I can't update with a text editor, I end up not doing the update. Anyway, I had this idea of firing up a remote desktop viewer and also my IDE when I get to work. When I was messing with Condi on my phone, I got the idea and it was very easy following the instruction from a superuser post.

The even that I was looking for was connectivity to my company's WIFI network. Even though I always connect to Ethernet cable anyway, this ensure that things would fire up even if I forgot to connect it. 

Applications and Services Logs > Microsoft > Windows > WLAN-AutoConfig > Operational

The event ID I was looking for was 8001. The commands I used were:

"C:\Program Files\VMware\VMware View\Client\bin\wswc.exe" -userName <my_username> -password <plaintext_omg> -unattended -serverURL <work.more.com> -desktopLayout multimonitor
"C:\Program Files (x86)\JetBrains\pycharm.lnk" H:\project
