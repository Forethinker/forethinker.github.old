  • Characters are escaped with a back trick: `: 0, a, b, n, r, t, ', "
  • Comparsion: lt, gt, eq, neq, le, ge, like
  • cd, cp, mv, rm, sort, less, more
  • home directory is $home
  • backtick outside of string is used to break up a command into multiple lines

File manipulation

  • Out-File, >: some frequently used options are -NoClobber, -Append, and Encoding. The output file is encoded in little endian unicode by default.
  • Add-Content, Set-Content: ASCII encoded, read-protect upon writing. Out-File vs Set-Content

Loop

ForEach-Object {$$.Property}. Or there is a syntax sugar %. $ - object that got passed in via pipe

For ($i=0;$i –le $Var.count; ++$i)

Sort

-Descending -Property

Help

Get-Help: -Online

Executing

.ps1 scripts must be ran with full path or .\ prefix to denote the current directory as the path. With quoted script names (to escape space characters), there must be & operand

Process Object

Start-Process procName : run in the background .WaitForExit() :

Get

Get-Command: list cmdlets -Name, -Module, Get-Process: list running processes Get-Service: list services running Get-WmiObject -Class: Win32_Product - installed software | Get-Member: get the members of the passed in object Get-Content or cat: $_ becomes an array with the index -Head #, -Tail #, -Wait Get-History: Get-Host: version information Get-Childitem: -recurse FullName, LastWriteTime Get-Date: -date: the default flag. -format takes both quick formatting and full formatting. -Uformat takes %j for day of the year and %V for the week of the year. Get-Content use -ReadCount 0 for a large file or use foreach to loop through the line and ditch the piepline altogether for the maximum performace. Get-Eventlog

select: -skip

Date

adddays tostring

Write

Write-Warning Write-Content Write-Host Write-Output

Creation

New-Item -Name, -Path

Retrival

Select-Object or select: filter with flags -Property | Where-Object {cond}: filter by condition. Works on a single object. For an array, ForEach-Object is needed. Compare-Object: compare objects -ReferenceObject (Get-Content %), -DifferentObject (Get-Content %))

Formatting

sort -Property, -Descending Format-List: Show properties as list Format-Table: Shows properties as a table (columnized) -AutoSize Format-Wide: list everything in column from the first property of the object, sorted alphabetically. -Property, -Column #: number of columns to display

Conditional

Get-Help About_If - -and, -or -not, ! - test-path -If ElseIf Else

Security

Get-ExecutionPolicy Set-ExecutionPolicy - Restrict: Default. No scripts can run - AllSigned: script must have a local digital signature - RemoteSigned: script must have a local or remote digital signature - Unrestricted: no digital signature is required

String

.Length .Substring(start, indexFromStart) -replace, - match, -nonmatch: for replace, `$1 can be used to match () grouped matches. .Split('=') .Trim()

Examples

Get a list of installed software on your pc

Get-WmiObject -Class Win32_Product | Select-Object -Property Name

Compare installed software on two different machines:

Compare-Object -ReferenceObject (Get-Content C:\Software\PCapps.txt) -DifferenceObject (Get-Content C:\Software\LAPTOPapps.txt)

Setting Up

For Powershell v. 3.0 you need Windows Management Framework 3.0 on a Windows 7 SP1 and .NET Framework 4. You need PS 3.0 in order to use commands like Update-Help or Invoke-WebRequest, but it still will not give you the full cmdlets that Windows 8 has. You can check which version of PowerShell you are running by invoking

Get-Host

-Set Execution Policy By default you cannot run any script on PowerShell. That sounds pretty dumb, but it is a good security measure considering that 99% of Windows Users will not use it. Set-ExecutionPolicy RemoteSigned is a good compromise for security and usability. -Create Powershell Profile aka .psrc New-Item –Path $Profile –Type File –Force

Map Network Drives

New-PSDrive –Name "K" –PSProvider FileSystem –Root "\\server\directory" –Persist

Where K is the driver letter

Display Path directories line by line:

 $a = $env:path; ; $a.Split(";")

execute all scripts in a directory

powershell.exe -noexit get-childitem &'c:\my scripts'

execute powershell in VBScript

Set objShell = CreateObject("Wscript.Shell") objShell.Run("powershell.exe -noexit c:\scripts\test.ps1")
