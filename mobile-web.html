Adaptive Design using Media Queries:

Break points are determined by viewports in Adaptive Design, so keep in mind the popular device width sizes (e.g. 768px)

<link rel="stylesheet" href="global.css" media="all" />
<link rel="stylesheet" href="main.css" media="screen" />
<link rel="stylesheet" href="paper.css" media="print" />
<link rel="stylesheet" href="mobile.css" media="handheld"/>

media property specifies which media the linked resource applies to. All CSS files referred by link tags are downloaded, bu only the CSS properties that meet conditions will be applied. This approach quickly diminished in practice, because they do not target many properties of a device very well.

Media Queries, therefore, was introduced as a part of CSS3 specification. The conditions are specified in css files in the following format: @media type (cond) {...}. Each conditions are wrapped in parenthesis and the conditions can have logical operators between them, which are not, only, and, or(,). Some popular conditions include: min-width, max-width, and orientation.

@media screen and (max-width: 320px) {
 body {
  font-size: 100%
 }
}

The font-size, border, width or float properties may be changed for a better viewing experience. It is a convention to put your media specific stylesheet on the bottom of your css file, for a better organization and load time.

Responsive Design

Responsive is superset of adaptive. Responsive design provides continuity between contexts, are portable, and accessible. Break points are decided by the contents. Other directives include: simplifying contents, prioritizing layout, and optimizing user experience.

Responsive Media

To make a image adjust to the screen instead of staying in a static size, first save a large sized media and set max-width property to adjust the size for the viewport. IE6 is not supported and for IE7&FF2 in Win7, the use of AlphaImageLoader is recommended.

Retina Images

In order to deal with pixelated images in retina displays, first save the larger version of media file with name@2x.png and apply it based on the device's pixel density. The ratio of 2 is optimal for retina, 1.5 for HD Screen of Android devices.

@media
only screen and (-webkit-min-device-pixel-ratio: 1.5),
only screen and (min-device-pixel-ratio: 1.5) {
 .logo {
   background-image: url(logo@2x.png);
   -webkit-background-size: 12px 16px;
   background-size: 12px 16px;
 }
}

This approach works with img, embed, object, video tags.

Other robust options include: - FitText to make font-sizing flexible - Lettering.js for more robust control over your typography - (FitVids.js)[http://fitvidsjs.com/] for flexible, responsive videos

In order to get around files being downloaded unnecessarily, PictureFill uses javascript to handle alternative image sizes.

<picture alt="Our Camp Site">
    <!-- Smallest size first - no @media qualifier -->
    <source src="camp-site.jpeg" />
    <!-- Large size - send to viewports 800px wide and up -->
    <source src="camp-site-lrg.jpeg" media="(min-width: 900px)" />
    <!-- Fallback content for non-JS or non-media-query-supporting-browsers -->
    <noscript>
        <img src="camp-site.jpeg" alt="our camp site" />
    </noscript>
</picture>
