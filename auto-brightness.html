Setting Auto-brightness with Calise and brightness.py

January 01, 2014 [ hack ] #automation Read in 6 min.

After following this tutorial you will have a laptop and an external monitor that changes brightness depending on the ambient brightness detected by a webcam. Requirements: a laptop with a webcam and linux OS (untested on a desktop with a webcam)

Calise

Calise takes care of ambient brightness using a webcam and this is all you really need if you do not have an external monitor.

sudo pip install pyephem
sudo apt-get install python-qt4 python-distutils-extra
cd ~/downloads/calise-*
sudo env python setup.py build
sudo env python setup.py install --install-layout=deb --prefix=/usr

You will get some errors about using Microsoft modules, ignore that.

You might also have to install following python modules, I had them installed already

sudo pip install dbus gobject2 xdg

Start by typing calise --configure and it will guide you through calibration process.

If you want to delete later, you can do the following:

sudo env python setup.py install --install-layout=deb --prefix=/usr --record touninstall.txt
cat touninstall.py | sudo xargs rm -rf

Brightness.py

Brightness.py controls the brightness of up to two monitors using xrandr underneath. I forked from lordamit and it seems as though the program has gotten pretty stable. My modification probably is not ready to serve all Brightness.py users as I only modified the scrip to control the secondary monitor (my external monitor) and let Calise take care of my primary monitor (laptop screen), but it serves me well and maybe for you too!

Here the link to the script on my github.

Autostart

I have the following lines in autostart file (I used Crunchbang OS), put these lines in an appropriate file for your OS that will be executed upon boot.

calised --capture-interval 30 &
(cd ~/Dropbox/linux/scripts && python brightness.py -c &)

The first line refreshed calise's brightness value every 30 seconds.

Don't use calise, because pm-utils tries to pause calised when going to sleep or hibernation mode. There is no pause option for calise

The research - though process behind brightness.py modification

I asked myself if the computer monitor's brightness is adjusted according to the ambient light, what is the best way to change the brightness? I thought of 3 criteria:

  1. The brightness adjustment is instantaneous / gradual
  2. Time interval is set to delay when the change occurs
  3. The brightness is leveled (like a laptop monitor, says 15 different levels) and brightness stays at (or gravitates, with gradual option from 1) at that level.

I think, from the usability point of view, option 1. gradual 2. ~10 sec time delay 3. 100 level is the best, like a Macbook monitor. Unfortunately, I found this difficult to achieve. Currently brightness.py changes brightness via xrandr and it lacks the direct hardware access. I am not sure if anybody out there who manages to change the brightness of an external monitor gradually, but even with 100 levels of brightness, I noticed some flickering. If I lower the delta of the change so that I have something like 500 or 1000 levels, the change is not noticeable, but I/O bus seems to get overloaded, resulting in slow mouse and keyboard responses.

So I settled for less. I modified brightness.py to change the monitor brightness every 2 minutes by default to any one of 16 levels set by calise. This ended up working well. calise changes my laptop brightness every 30 seconds, this lower interval is useful when I am moving around, but not so much when I have my big monitor and I am forced to be stationary. brigthness.py polls, but I think this was a simpler approach than using, say pynotify or watchdog, since I am dealing with only one file for a single integer value. CPU usage is unnoticeable.

In brightness.py there is an option to supply -o, --offset, which defaults to 20. This is to help balance the difference between a laptop screen and an external monitor. You can see the effect it has on setting the brightness level by setting the -vv --verbose option and seeing the numerator and denominator values. Without this offset, I found my external monitor to be dark.

Lastly, if you want a real challenge, you can go for OpenCV, which I attempted to use at first, but it is quite bit of work. Brightness.py

Windows - BLUntrl

If you use Windows, you are not out of luck. Many computer manufacturers include auto-bright features for Windows 8. And if you use Windows 7 and below, you can use BLUtrl The author kindly supplied a tool that hides the application from the user's view. http://www.expocenter.com/hideit/

Building BLUntrl

The above link as the binary, but because BLUntrl changes brightness too quickly, I wanted to put some threshold to prevent it from flickering. So I had to change the source code to get that job done. I really hate Visual Studio with passion, just because it does not have search box for the options, but I happened to have Microsoft Visual Studio Professional 2012. The following was tested on Windows 7 x64. First Install Windows SDK Change "Configuration Properties->General->TargetName", because by default it wants to build as BLUntrl, but the project name is not set up that way. Just go with the instruction I have and rename the file back you preference later.

$(ProjectName) -> videoInputBasicDemo

Change "Configuration Properties->Linker->General->Output File" to $(OutDir)videoInputBasicDemo.exe Set "Configuration Properties->General->Target Name" to videoInputBasicDemo Add the following directory to "Configuration Properties->C/C++->General->Additional Include Directories"

C:\Program Files\Microsoft SDKs\Windows\v7.1\Include

Add the following directory to "Configuration Properties->VC++ Directories->Library Directories"

C:\Program Files\Microsoft SDKs\Windows\v7.1\Lib
