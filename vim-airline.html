Vim-Airline is here!

September 17, 2013 [ hack ] #vim Read in 6 min.

Airline is the latest and hottest putting skins on statusline. There are at least a dozen of websites that explains how to install powerline fonts, but I found them confusing and felt like I was dealing with a bad customer service ("follow this link"). So here is how I got vim-airline to work with powerline fonts.

John Chain wrote about why airline is better. In short, vim-arline uses less code (1/12th of powerline, 1/3 of vim-powerline) and it is not python dependent. On a side note, I did not have to follow his advice on set lazydraw to get airline working in terminal.

My intention was to use Consolas font everywhere along with airline (vim, gvim, cygwin, windows, linux). I was successful, mostly. I still could not get powerline symbols to show up in cygwin terminal. Also git-gutter's color coding got lost whenever I saved a file, but I am hoping it will get straightened out over time.

Install vim-airline

I have tried Pathogen, NeoBundle, and Vundle. I recommend Vundle. I found it must faster in updating vim plugins.

# in .vimrc
Bundle 'blind/vim-airline'
# on commandline
:BundleInstall!

Font installation in Windows

I do not use any of these fonts, skip to Consolas Font section if you are a fan of that font.

Download the font:

git clone https://github.com/Lokaltog/powerline-fonts

If you right click them, there should be install option in the menu. Or you can drag it straight to Windows Fonts folder, or you really ought start using NexusFont to manage your font. Anyway, here are my opinions of the font.

Anonymous
DejaVuSansMono - 2nd best looking font
DroidSansMono - 2nd worst looking font
Inconsolata
InconsolataDz
MenloforPowerline - the worst looking font
MesloforPowerlineDZ
SourceCodePro 
UbuntuMono - the best looking font

All these fonts will be installed with 'Powerline' string as the suffix to their names. 'DejaVuSansMono Powerline's regular font (it shows up as Book in the font settings box) was not showing up as an option, forcing the user to choose 'Bold' or 'Bold Italic'. I was able to get the regular fonts back by deleting the 'DejaVuSansMono' font.

Font installation in Linux

Once again, skip to Consolas Font section if you are a fan of that font. I use Crunchbang Linux, so the installation details may differ. The installation details from Powerline Documentation says that 'there are two ways of installing the required fonts'. I think it is meant to says two STEPS, because they go together. Here is what I did.

cd ~/.fonts
git clone https://github.com/Lokaltog/powerline-fonts
wget https://github.com/Lokaltog/powerline/raw/develop/font/PowerlineSymbols.otf
wget https://github.com/Lokaltog/powerline/raw/develop/font/10-powerline-symbols.conf
# Merge the contents of 10-powerline-symbols.conf to ~/.fonts.conf
fc-cache -vf ~/.fonts

I say it must be two STEPS, because if you look at the content of '10-powerline-symbols.conf', it is patching 'Powerlinesymbols.otf' to Lokaltog's fonts and some of the popular linux fonts. The article does not make it clear that first way is a prerequisite for the second way, but it is the only explanation on the web I found, so I am very grateful. You can see the copy of my modified .fonts.conf here.

Consolas Font

Consolas is my favorite font, because it is a popular choice and I have no spine. I kid. Consolas font has design that is simple and clear. It is very comfortable to read. It is as if the font is trying its best to hide away complexity and let you focus on what you actually see. Euguneching was the original author of consolas-powerline-vim, a font patched up for powerline. When I tried it back in 2012, it did not work for me. So I gave up on using powerline altogether. That is how much consolas means to me. I even tried using fontpatcher on a copy of consolas font from Windows, but it still would not work. But I found Runsisi's version of font this time around and it worked well (Big dot next to trailing line notifcation in Windows gVim, does not show up though, sadly). I do not know what kind of magic he pulled off, but font designing is something that I will never do, so I am comfortable with not knowing the underlying details. And no, I will not bother fixing that one glyph in Windows gVim.

The step for Linux is similar to that of above:

cd ~/.fonts
git clone https://github.com/runsisi/consolas-font-for-powerline
fc-cache -vf ~/.fonts

Airline in Action

You need to set font encoing to 'utf-8' for glyphs to display properly. In gVim, that means setting set encoding=utf-8; In terminals, going to the preference and change the appropriate settings. For cygwin, it should be done already, but it can be changed in 'cygwin.bat' file in the cygwin installation directory. set LANG=en_US.UTF-8

Then I have the following lines in my .vimrc. This allows me to set the consolas font for Windows gVim and Linux gVim, respectively.

if has ('win32')
    set guifont=Powerline_Consolas:h9:cANSI
else
    set guifont=Powerline\ Consolas\ 10
endif

Airline's tabline is really well designed. It is turned off by default, but I really recommend the readers to use it. I used to have a problem where the tab separator would not show clearly in my Windows gVim. That problem is not gone thanks to Airline's tabline. Airline's themes come included in the git repo. The screenshots are also available online.. I think powerlineish theme looks the best.

let g:airline#extensions#tabline#enabled = 1
let g:airline_theme             = 'powerlineish'

Cygwin terminal

As I mentioned before, the symbols do not work. But it is very easy to turn on the non-glyph mode. Because glyph is turned off by default you just enable it with GUI mode.

if has ('win32unix') && !has('gui_running')
    let g:airline_powerline_fonts = 1
endif

Conclusion

Airline makes vim so beautiful. I get butterflies and chills as I stare into my vim now. I used to think that vim's vanilla version of statusline was enough since it shows all the information I need. Boy I was wrong. Color conveys information more quickly and efficiently than one may expect. I now believe that color encoding can be improved, always. Although color design is not a task I would pursue, it is something that I will keep tab on what pros are doing.
