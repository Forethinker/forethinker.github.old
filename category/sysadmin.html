Sendmail on Ubuntu
August 26, 2014 [ sysadmin ] Read in 1 min.

There are several options available for sending emails from your server: SMTP, postfix, and sendmail. I choose sendmail because it is the easiest to setup and I only need it to send out emails.

Changing hostname temporarily for now

This is the name that will end up after @ character in the email. It is also the name that shows up on your command shell prompt. 

hostname NEW_HOST_NAME

Your prompt will not reflect the change until you reboot.

Changing the hostname for when the system boots:

/etc/hosts

127.0.0.1 myhostname localhost.localdomain localhost

Configuration

Just say yes. 

sudo sendmailconfig
service sendmail restart

MAIL_HUB and LOCAL_REPLY

These are for unqualified and local names. /etc/mail/sendmail.mc

define(`MAIL_HUB', `example.com.')dnl
define(`LOCAL_RELAY', `example.com.')dnl

Test

-t flag i

echo -e "To: user@example.com\nSubject: Test\nTest\n" | sendmail -t -v

Install smaba and configure

sudo apt-get install samba
sudo vim /etc/samba/smb.conf

Configuring smb.conf

security = user

[homes]
   comment = Home Directories
   browseable = yes
   valid users = %S

valid users = %S means that only you, or anyone with your login account, can connect to your own Home folder via Samba.

SSH recipes
January 25, 2014 [ sysadmin ] #shell Read in 2 min.

SSH keyless entry does not work

Check the logs on /var/log/auth/log or /var/log/secure (CentOS). You probably have a permission issue.

chmod 700 ~/
chmod 700 ~/.ssh/
chmod 600 ~/.ssh/authorized_keys

make sure the options are set properly on /etc/ssh/sshd_config - it needs root permission

RSAAuthentication yes
PubkeyAuthentication yes
AuthorizedKeysFile      %h/.ssh/authorized_keys

then restart

sudo service sshd restart

Diff files over remote

ssh <remote-host> "cat /path/to/remotefile" | diff - /path/to/localfile

Copy over SSH key so you don't have to type password every time:

sshk() {
    ssh-copy-id -i ~/.ssh/id_rsa.pub "$@"
}

pdbedit to configure samba on linux machine
December 02, 2013 [ sysadmin ] #samba Read in 1 min.

Group

Create a group

gropuadd groupName

Delete a group

groupdel groupName

Users

Create a user

useradd -m -g groupName -s /sbin/nologin userName
do

-m create home, -g specify group, -s' specify shell

Add User to samba and create a new password:

pdbedit -a -u userName

Remove User from samba (modify the user by adding a new one):

pdbedit -x -u userName
