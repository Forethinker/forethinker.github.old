Ironing out flash issues on Google Chrome
March 23, 2016 [ hack ] Read in 2 min.

Flash plugin was not working and after ticking all possible options in chrome://settings, chrome://flags, chrome://components, and chrome://plugins, I still was not able to make it work. Okay, so even html5 video was not working for me and that turned out to be video driver issue (!). As for PepperFlash. I had to rename directory C:\Users\john\AppData\Local\Google\Chrome\User Data\PepperFlash to something else so that I can a fresh copy downloaded via chrome://components.

So with the absence of NPAPI, some plugins that took advantage of flash plugin's security hole is gone. For me I used Enounce's myspeed to speed up the playback speed of flash videos.

Fortunately there is a chrome plugin that could control the playback speed of HTML5 videos, which is the new standard on the web anyway. It would have been nice if the plugin for forcing html5 also work so that the chrome web browser tries html5 before falling back to flash. But this is not working for me. I chose to disable flash player altogether since I don't really use it.

NPAPI is also going to get ditched by Firefox and fortunately there is addon to control the playback speed of html5 vidoe here.
