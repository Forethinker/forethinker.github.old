Useful packages

They are pretty much all useful. These are the settings I put in grunt.initConfig({})

wiredep

First add these meta tags to index.html

<head>
    <!-- bower:css -->
    <!-- endbower -->
    <!-- bower:js -->
    <!-- endbower -->
</head>

in gruntFile.js:

grunt.loadNpmTasks('grunt-wiredep');

then in grunt.initConfig:

wiredep: {
  target: {
    src: 'index.html' // point to your HTML file.
  }
}

and execute it by:

grunt wiredep

Nginx is an event-driven file server and it is much lighter than apache in resource usage.

sudo service nginx start update-rc.d nginx defaults

There could be two configuration files, but similar to apach, one typically sources another.

https://www.digitalocean.com/community/tutorials/how-to-optimize-nginx-configuration

worker_processes:

grep processor /proc/cpuinfo | wc -l

worker_connections:

ulimit -n

override worker_connections

events {
    worker_connections  19000;  # It's the key to high performance - have a lot of connections available
}
worker_rlimit_nofile    20000;  # Each connection needs a filehandle (or 2 if you are proxying)
